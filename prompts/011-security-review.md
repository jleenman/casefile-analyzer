# Prompt 011 - Security Review

## Doel

Voer een security- en privacyreview uit op de MVP-architectuur en eventuele code.

## Context

De tool verwerkt potentieel zeer gevoelige dossiers. Demo's moeten veilig blijven.

## Inputbestanden

- `docs/privacy-security.md`
- `docs/risks-and-ethics.md`
- `docs/architecture.md`
- Eventuele code en configuratie.

## Outputbestanden

- Bijgewerkte securitydocumentatie.
- Issue- of TODO-lijst voor risico's.
- Kleine fixes waar veilig en afgebakend.

## Acceptatiecriteria

- Publieke opslagrisico's zijn gecontroleerd.
- Geen secrets in repository.
- Echte data wordt niet gebruikt in tests of demo's.
- Privacy, bronverwijzingen en controleerbaarheid zijn leidend.
- Eindig met een kleine commit of duidelijke wijzigingssamenvatting.
